Create `deploy/` directory with two systemd unit files.

**deploy/steam-sniper-dashboard.service:**
```ini
[Unit]
Description=Steam Sniper Dashboard (FastAPI)
After=network.target

[Service]
Type=simple
User=root
WorkingDirectory=/opt/steam-sniper
ExecStart=/root/.local/bin/uv run python server.py
Restart=on-failure
RestartSec=10
EnvironmentFile=/opt/steam-sniper/.env

[Install]
WantedBy=multi-user.target
```

**deploy/steam-sniper-bot.service:**
```ini
[Unit]
Description=Steam Sniper Telegram Bot
After=network.target

[Service]
Type=simple
User=root
WorkingDirectory=/opt/steam-sniper
ExecStart=/root/.local/bin/uv run python main.py
Restart=on-failure
RestartSec=10
EnvironmentFile=/opt/steam-sniper/.env

[Install]
WantedBy=multi-user.target
```

Key details:
- Both use `EnvironmentFile` pointing to `/opt/steam-sniper/.env` (not dotenv load)
- `uv run` resolves dependencies from pyproject.toml in WorkingDirectory
- `RestartSec=10` prevents restart storms
- `Restart=on-failure` + `WantedBy=multi-user.target` ensures DEPLOY-04 (autostart on reboot)
- Both share WorkingDirectory so SQLite path `data/sniper.db` resolves to same file (DEPLOY-03)
- Use full path `/root/.local/bin/uv` for uv binary (standard uv install location for root)

**Deploy steps (executed via paramiko SSH):**

1. **Connect** â€” `paramiko.SSHClient()`, `AutoAddPolicy`, connect with `look_for_keys=True` (uses SSH agent or `~/.ssh/id_rsa`)
2. **Ensure uv installed** â€” `command -v uv || curl -LsSf https://astral.sh/uv/install.sh | sh`
3. **Create remote dirs** â€” `mkdir -p /opt/steam-sniper/data`
4. **Upload project files** â€” SFTP put each file to `/opt/steam-sniper/`
5. **Upload systemd units** â€” SFTP put .service files to `/etc/systemd/system/`
6. **Install deps** â€” `cd /opt/steam-sniper && /root/.local/bin/uv sync`
7. **Reload systemd** â€” `systemctl daemon-reload`
8. **Enable + restart services** â€” `systemctl enable --now {SERVICE_DASHBOARD} {SERVICE_BOT}` then `systemctl restart {SERVICE_DASHBOARD} {SERVICE_BOT}`
9. **Verify** â€” `systemctl is-active {SERVICE_DASHBOARD}` and `systemctl is-active {SERVICE_BOT}`, print status
10. **Print summary** â€” "Dashboard: http://194.87.140.204:8100", service statuses

**Error handling:**
- Wrap SSH commands in a helper `run(client, cmd)` that logs stdout/stderr and raises on non-zero exit
- Print each step as it executes (numbered log)
- If any step fails, print error and exit (don't continue half-deployed)

**SSH key resolution:**
The script should try paramiko's `connect(look_for_keys=True)` first. If that fails, look for key at `~/.ssh/id_rsa` or `~/.ssh/id_ed25519`. This matches how the user connects to VPS from Windows.
