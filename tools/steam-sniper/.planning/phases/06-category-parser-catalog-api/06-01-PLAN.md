Add a new endpoint to server.py:

1. Add import at top of server.py:

2. Add `GET /api/catalog` endpoint with these query parameters:
   - `limit: int = Query(default=50, ge=1, le=200)` — items per page
   - `offset: int = Query(default=0, ge=0)` — pagination offset
   - `category: str | None = Query(default=None)` — filter by category (e.g. "knife", "rifle")
   - `sort: str = Query(default="name_asc")` — sort option, one of: "name_asc", "name_desc", "price_asc", "price_desc", "count_desc"
   - `q: str | None = Query(default=None)` — search query (substring match on item name, case-insensitive)

3. Endpoint implementation:
   ```python
   @app.get("/api/catalog")
   @beartype
   def get_catalog(
       limit: int = Query(default=50, ge=1, le=200),
       offset: int = Query(default=0, ge=0),
       category: str | None = Query(default=None),
       sort: str = Query(default="name_asc"),
       q: str | None = Query(default=None),
   ) -> dict:
   ```

   Logic:
   a. Build list from `_prices.values()`, enriching each item with `category` field via `classify(item["name"])` and `price_rub` via `item["price"] * _lis_rate()`.
   b. If `category` param provided: filter to only items matching that category.
   c. If `q` param provided: filter to items where `q.lower()` appears in `item["name"].lower()` (all words must match, split by space — same logic as existing /api/search).
   d. Sort using `sort` param:
      - "name_asc": alphabetical by name (default)
      - "name_desc": reverse alphabetical
      - "price_asc": cheapest first
      - "price_desc": most expensive first
      - "count_desc": most available first (for "in stock" browsing)
   e. Record `total` count AFTER filtering but BEFORE pagination.
   f. Slice with `offset` and `limit`.
   g. Return:
      ```json
      {
        "items": [
          {
            "name": "AK-47 | Redline (Field-Tested)",
            "category": "rifle",
            "price_usd": 12.34,
            "price_rub": 1234.56,
            "count": 5,
            "url": "https://lis-skins.com/...",
            "available": true
          }
        ],
        "total": 24000,
        "limit": 50,
        "offset": 0,
        "categories": {"knife": 1200, "rifle": 3400, ...}
      }
      ```
   h. The `categories` field is a dict of category -> count for ALL items (ignoring current filters). This powers the sidebar category counts in Phase 8. Cache this in a module-level variable `_category_counts` that gets rebuilt on each `_collect_once()` call.

4. In `_collect_once()` function, after `_prices` dict is rebuilt (line ~112), add category count computation:
   ```python
   global _category_counts
   _category_counts = {}
   for item in _prices.values():
       cat = classify(item["name"])
       _category_counts[cat] = _category_counts.get(cat, 0) + 1
   ```
   Add `_category_counts: dict[str, int] = {}` to module-level state (near line 41).

5. The `available` field in response items: `item["count"] > 0`.

Do NOT modify any existing endpoints. Only add the new endpoint and the import + category count computation.
