**Complete `<script>` rewrite.** Remove ALL existing JS (lines 551-816). Replace with new JS that uses API endpoints. Key functions:

1. `async function init()` -- calls loadStats(), loadWatchlist(), loadAlerts(). Sets up 5-min interval via `setInterval(refresh, 300000)`.

2. `async function refresh()` -- calls loadStats(), loadWatchlist(), loadAlerts() (the auto-refresh cycle, no page reload).

3. `async function loadWatchlist()` -- `fetch('/api/watchlist')`, parse JSON. Store `usdRub` from response. Render sell and buy sections using `renderTable()`. Update header stats (usdRate, totalItems, lastUpdate) from response fields `usd_rub`, `total_items_lis`, `updated_at`.

4. `function renderTable(entries)` -- Build the watchlist table HTML. For each entry use response fields directly:
   - Name: `entry.name` with link to `entry.url`
   - Price: `entry.current_price_rub` formatted as "X,XXX ₽" plus `entry.current_price_usd` as "$X.XX" in `.sub` span
   - Delta: `entry.delta_pct` with up/down class
   - Target: `entry.target_rub` with type badge (buy/sell)
   - Qty: `entry.qty`
   - Delete button: `<button class="wl-remove" data-name="${entry.name}">✕</button>` with click handler
   - Row gets `data-item-name="${entry.name}"` attribute (for chart click in 03-02)
   - Row gets `cursor: pointer` style and click handler that will be used by 03-02 for chart display

5. `async function removeItem(name)` -- `fetch('/api/watchlist/' + encodeURIComponent(name), {method: 'DELETE'})`. On success, call loadWatchlist().

6. `async function loadStats()` -- `fetch('/api/stats')`. Update hero block: heroValue shows `total_value_rub` formatted as "XXX,XXX ₽", heroDelta shows `delta_pct` with +/- sign and green/red class, heroCount shows `total_items`.

7. `async function loadAlerts()` -- `fetch('/api/alerts')`. Render into alertsList. Each alert: `.alert-item` div with message text and relative time. If no alerts, show "No alerts yet" dim text.

8. Search functions: `doSearch()` uses `fetch('/api/search?q=' + encodeURIComponent(q))`. Parse response.results array. Each result shows `result.name`, `result.price_rub` formatted as RUB, `result.count`, and "+ Add" button calling `openModal(result)`.

9. `function openModal(item)` -- receives the search result object `{name, price, price_rub, count, url}`. Populates modal fields. Sets `pendingItem = item`.

10. `async function addToWatchlist()` -- reads modal form values. `fetch('/api/watchlist', {method: 'POST', headers: {'Content-Type': 'application/json'}, body: JSON.stringify({name: pendingItem.name, type, target_rub, qty})})`. On 201 success: closeModal(), clear search, loadWatchlist().

11. `function closeModal()` -- hide modal overlay, clear pendingItem.

12. Event listeners: search input with 200ms debounce, click-outside to close search dropdown, Escape to close modal, delegated click on `.wl-remove` buttons for delete.

**Critical details:**
- NO localStorage usage. All data from API.
- NO direct fetch to lis-skins or CBR URLs. Everything through /api/*.
- All prices display in RUB (the API already returns current_price_rub).
- USD price shown as secondary in smaller text.
- Format RUB: `Math.round(val).toLocaleString('ru-RU') + ' ₽'`
- Format delta: `(val > 0 ? '+' : '') + val.toFixed(1) + '%'`
- Use event delegation on watchlist table for delete and row click.
- Search debounce: 200ms timeout on input event.
- Empty states: "Watchlist is empty" for no items, "Nothing found" for empty search, "No alerts yet" for empty feed.
