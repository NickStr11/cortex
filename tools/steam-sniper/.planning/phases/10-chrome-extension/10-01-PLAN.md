**extension/background.js -- Service worker handles API calls:**

Architecture: content script sends message -> background worker does fetch -> returns result.
This is REQUIRED because content scripts in MV3 run in the page origin (lis-skins.com HTTPS) and cannot fetch HTTP APIs (mixed content block). Background service worker has `host_permissions` and can fetch any permitted origin.

```
const API_BASE = "http://194.87.140.204";
const DEFAULT_USER = "lesha";
```

Listen for `chrome.runtime.onMessage`. Expected message format:
```json
{
  "action": "addToList",
  "item_name": "AWP | Asiimov (Field-Tested)",
  "list_type": "favorite"
}
```

On message:
1. `fetch(API_BASE + "/api/lists", { method: "POST", headers: {"Content-Type": "application/json"}, body: JSON.stringify({user: DEFAULT_USER, item_name: msg.item_name, list_type: msg.list_type}) })`
2. Parse response JSON
3. `sendResponse({ok: true})` or `sendResponse({ok: false, error: "..."})`
4. CRITICAL: return `true` from onMessage listener to indicate async response (otherwise Chrome closes the message channel before fetch completes)

**extension/content.js -- DOM injection on lis-skins.com item pages:**

Lis-skins.com DOM is undocumented. The content script must be defensive and discover item names from multiple possible sources. Strategy:

1. **Page detection**: Check if URL matches item page pattern. Lis-skins item pages have URLs like `https://lis-skins.com/market/csgo/ITEM-SLUG/` or similar patterns with `/market/` in the path. Also check for the presence of item-specific DOM elements.

2. **Item name extraction** (try in order, use first that works):
   a. `document.querySelector('h1')` -- most item pages have the item name in h1
   b. `document.querySelector('.market-item-name, .item-name, [class*="item-name"], [class*="ItemName"]')` -- common CSS class patterns
   c. `document.title` -- strip suffix like " - lis-skins.com" or similar
   d. If none found, do NOT inject button (defensive)

3. **Button injection**: Create a fixed-position container in bottom-right corner with:
   - Main button "Add to Sniper" (orange, matches dashboard theme #e67e22)
   - On click, show dropdown with two options: "Favorite" and "Wishlist"
   - Each option sends `chrome.runtime.sendMessage({action: "addToList", item_name, list_type})`
   - On response: show toast notification (green for success, red for error) that auto-hides after 3 seconds

4. **Duplicate prevention**: Check `document.querySelector('.sniper-btn-container')` before injecting. Content scripts can re-run on SPA navigation.

5. **SPA handling**: Lis-skins may use client-side routing. Set up a `MutationObserver` on document.body or use `setInterval` (every 2 seconds, max 5 checks) to detect if the page content changed (new item page loaded without full reload). Re-inject button if URL changed.
