**server.py — add trend data to /api/catalog response:**

In the `get_catalog` endpoint, after building each item dict in the loop, add trend data for case items. Modify the item dict construction to include trend info:

```python
# Inside the loop in get_catalog, after building item dict:
item_dict = {
    "name": item["name"],
    "category": cat,
    "price_usd": item["price"],
    "price_rub": round(item["price"] * rate, 2),
    "count": item.get("count", 0),
    "url": item.get("url", ""),
    "available": item.get("count", 0) > 0,
}
# Add trend for case items (used by cases tab)
if cat == "case":
    item_dict["trend"] = _calc_trend(item["name"].lower())
items.append(item_dict)
```

This adds a `trend` field `{direction: "up"|"down"|"flat", pct: float}` only to case items (not all 24k items — trend calculation queries DB history, expensive for 24k).

**static/js/cases.js — new module:**

Create `cases.js` that fetches `/api/catalog?category=case` and renders a card grid with trend badges. Follow the catalog.js pattern but simpler (no sidebar, no category filter — cases tab IS the filter).

Module state:
- `_sort = 'name_asc'` (default sort)
- `_query = ''` (search within cases)
- `_offset = 0`, `_limit = 50`
- `_loaded = false` (lazy load on first tab activation)
- `_searchTimeout = null`

Imports: `fmtRub` from utils.js, `events` from events.js, `cacheCatalogItems` from lists.js

Export `initCases()` and `loadCases()`.

`loadCases()`:
1. Build URLSearchParams: limit, offset, sort, category='case', q (if _query set)
2. `fetch('/api/catalog?' + params)` 
3. Call `renderCasesGrid(data.items)` and `renderCasesPagination(data.total, data.offset, data.limit)`
4. Call `cacheCatalogItems(data.items)` to enable list toggles
5. Emit `events.emit('cases:loaded', { total: data.total })`

`renderCasesGrid(items)`:
- Target: `document.getElementById('casesGrid')`
- If empty: show "No cases found" empty state
- For each item, render a `.cat-card` with:
  - Image (same STEAM_IMG_BASE pattern as catalog.js)
  - Item name (`.cat-card-name`)
  - Price in RUB (`.cat-card-price`)
  - Availability count (`.cat-card-count`, add `.unavailable` class if count=0)
  - **Trend badge**: if `item.trend` exists, render a `<span class="trend trend-{direction}">` with arrow + percentage. Format: "up" = `+X.X%`, "down" = `-X.X%`, "flat" = "0.0%". Use existing `.trend`, `.trend-up`, `.trend-down`, `.trend-flat` CSS classes (already in styles.css from watchlist cards).
  - List toggle buttons (heart + star) using same data attributes as catalog.js: `data-list="favorite"` / `data-list="wishlist"`, `data-name="{safeName}"`. This lets lists.js handle clicks via existing event delegation.

`renderCasesPagination(total, offset, limit)`:
- Target: `document.getElementById('casesPagination')`
- Reuse same pagination logic as catalog.js `_buildPageRange` — copy the function locally or extract shared. Since it's ~20 lines, copy locally to keep modules independent.

`initCases()`:
1. Pagination click delegation on `#casesPagination`
2. Sort dropdown change on `#casesSort`
3. Search input with 400ms debounce on `#casesSearch`
4. Lazy load: listen for hashchange, if `location.hash === '#cases' && !_loaded`, set `_loaded = true` and call `loadCases()`
5. Check on init if already on cases tab
