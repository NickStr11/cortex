## 1. Personal OS

Персональная операционная система — **набор нишевых микрорешений** (skills, rules, MCP), не монолит. Агент живёт в твоей среде и смотрит на твои файлы.

**Ключевой сдвиг**: раньше мы загружали контекст в GPT. Теперь агент приходит к нам.

**Определение агента** (управленческое):

| Компонент | Что это |
|-----------|---------|
| Задача | Промт или серия промтов |
| Ресурсы | Контекст, MCP-серверы, скиллы, данные |
| Полномочия | Чтение, запись, изменение, поиск |

> "Агента нельзя не создать. Если открыл Claude Code — там уже живёт агент." — FOS #16

## 2. Context Engineering

### Определение
> "Контекст — это информация, которая нужна, чтобы помочь именно тебе, в этот момент времени, целенаправленно."

### "Алхимия контекста"
Комбинирование кусочков под задачу (как Minecraft crafting):
- цели + разговор с боссом + шаблон = персонализированный онбординг
- **Не перегружай агента** — давай ровно то, что нужно

### Контекст о себе (~9-10 документов)
| Тип | Примеры | Обновление |
|-----|---------|------------|
| Статические | Личность, принципы, стиль работы | Раз в 1-2 месяца |
| Динамические | Текущие цели, приоритеты | Еженедельно/ежемесячно |

- Макс 10 документов, 2-3 страницы каждый
- Источники: терапия, тесты личности, рефлексия, NotebookLM для сжатия
- Альтернатива: один файл `aboutme.md` уже даёт другой уровень (FOS вебинар)

### Всё есть контекст
Скриншот, фото афиши на стене, голосовое сообщение, транскрипт звонка — всё оцифруется и передаётся модели. Границы расширились: визуал, аудио, текст.

### Навыки 2026 года (цитата продакта Google)
1. **Context curation** — курирование контекста
2. **Taste** — вкус (человечный)
3. **Agent orchestration** — оркестрация
4. **Judgment** — суждение
5. **Problem shaping** — формулирование проблем (не solving!)

## 3. Архитектура

### Skills → Rules → MCP → Context
```
Skills (короткие команды/abilities)
  → Rules (markdown-документы, tool-agnostic)
    → MCP Services (внешние сервисы)
      → Context (файлы о себе, проекте, команде)
```

### Структура скилла (FOS #16 Рис)
```
skills/
  digest-vibe-coders/
    skill.md          # Логика на естественном языке
    db_connection.py   # Детерминированная часть
    get_messages.py    # API-скрипт
    output_template.md # Шаблон вывода
```

### Уровни скиллов (FOS #21)
- **Глобальные** — видны всем сессиям
- **Командные** — через symlink из `.claude/` на Obsidian Sync
- **Проектные** — внутри конкретного репо

### Ключевые принципы
1. **Markdown = универсальный формат** — tool-agnostic, переносимый
2. **Composable** — один скилл в нескольких процессах
3. **Субагенты** — Haiku для bulk, Opus для reasoning
4. **Single Source of Truth** — Linear/задачи, Obsidian/заметки, Telegram/ссылки
5. **Multi-interface** — терминал, Telegram-бот, Obsidian, десктоп
6. **PostgreSQL > markdown для данных** — "через 3 недели markdown превращается в помойку" (FOS #16 Рис)

### Правило создания скиллов
> "Если повторяешь flow несколько раз — превращай в скилл." — FOS #16 Рис

## 4. Рабочие процессы

### Morning Review (~4 мин) — FOS #16
1. Запуск скилла `Today`
2. Агент спрашивает настроение
3. Параллельно: задачи (Linear) + встречи (Calendar)
4. Скоринг по Naval Ravikant (деньги, рычаг, видимость)
5. Выбирает **ровно 3 задачи** на день
6. Фокус дня → Obsidian

### Evening Review (~7 мин) — FOS #16
1. Скилл `Evening`
2. **Network Review** (Haiku): все переписки Telegram за день
3. Извлекает **обещания** → новые задачи ("Мария ждёт")
4. Опрос: как прошли 3 задачи?
5. Карточки людей → Obsidian

### Jarvis Start (инициализация) — FOS #21
1. `/jarvis-start` при каждом запуске Claude Code
2. Проверяет: кто он, дата, откуда запущен (сервер/ноутбук)
3. Загружает контекст: кто я, скилы, незавершённые задачи
4. Проверяет MCP-серверы и версию Claude Code

### Expert Panel (по Карпатому) — FOS #16
- Роли под ситуацию: CFO + Devil's Advocate + Therapist
- Каждая роль с доступом к глубокому контексту
- **Принцип, не копия** — адаптируй под себя

### Content Pipeline (Links Process) — FOS #16
1. Saved Messages в Telegram → inbox
2. Stage 0: скачивание (YouTube → транскрипт, LinkedIn → текст)
3. Stage 1: summary + концепции + люди
4. **Triage (агент-судья)**: защита плотности базы знаний
5. Одобренное → Obsidian

### Task Linker / NSWR — FOS #16 + #21
- Каждый ответ привязан к задаче из Linear
- Deep link + предложение обновить статус
- Решает разрыв между "херачу в терминале" и "трекинг задач"

### Interview (планирование) — FOS #21
- `/interview` поднимает 20-30 уточняющих вопросов перед задачей
- Долгое планирование → задача за 2-3 промта

### Research-first паттерн — FOS #16 Рис
- Каждый запрос начинать с исследования через exa.ai MCP
- Вставлять текущую дату (модель думает что 2024)
- "Используя официальные источники"

### Видео-пайплайн — FOS #16 Рис
1. Загрузка видео → извлечение аудио → Whisper (локально, ~20 мин/2ч)
2. Генерация субтитров → исправление терминов (отдельный скилл)
3. SEO-оптимизация → трекинг-ссылки → публикация
4. Результат: с 3-4 часов → ~25 минут

### Continuous Learning — FOS #21
- Если действие повторяется 2-3 раза → агент записывает в память
- Скилл `/continuous-learning` в конце сессии
- Следующий раз не тратит токены на повторение

## 5. Context Management

### 3 режима (FOS #21)
1. **Full Power** — полный контекст, все возможности
2. **Slow Mode** — компактный режим при 150-180K токенов
3. **Hand-off** — прекращение, подготовка handoff для следующей сессии

### Инфокомпрессор — FOS #21
- Скилл, ужимающий информацию на ~70% без потерь
- Сокращения, таблицы, упрощение грамматики

### Кастомный `/context-compact` — FOS #21
- Вместо дефолтного compact
- Записать всё в память → извлечь полезное → переехать в новый чат

### Логирование сессий — FOS #21
- Все сессии Claude Code → Obsidian
- Короткие сессии (<10 сообщений) → OpenRouter для summary
- Поиск через Raycast с кэшированием

### Remind — FOS #21
- `/remind 2h` → сессия восстанавливается через Mac Reminders

## 7. Контент и маркетинг с AI

### Трёхуровневая модель контента — FOS #17
```
Уровень 1: СМЫСЛЫ — кто я, миссия, контент-кор, грани
Уровень 2: ФОРМЫ — пиллары, визуальный код, ToV, форматы
Уровень 3: АССИСТЕНТЫ — боли аудитории, тексты, визуал, автоматизация
```

### 20 минут на контент (когда система настроена) — FOS #17
1. Тема → LLM с готовым промптом → структура
2. Варианты заходов (провокация, вина, нейрофизиология) → выбрать
3. Визуал: референсы из Pinterest → бот → обложка с текстом
4. Анимация + текст поста параллельно

### Вайб-маркетинг (0 бюджета) — FOS #19
1. Утренний парсинг Threads/Reddit/X
2. Генерация контента → 3 этапа критиков
3. Автодеплой в очередь
4. Хайп = трафик: быстро встраиваться в тренды (антипод, пародия)
5. **Аудитория важнее продукта** — spray-подход к гипотезам

### Минимальный стек контент-мейкера — FOS #17
- 1 LLM + 1 генератор изображений = 99% задач

### Telegram-боты для контента — FOS #17
- БананаБот — генерация визуала через API
- БайкерБот — коуч, который пинает делать контент

## 8. Инфраструктура

### VPS как центральный сервер — FOS #19 + #21
```
Personal VPS
  ├── Claude Code / OpenClaw (агент)
  ├── Obsidian (Docker)
  ├── Codex (кодинговый агент)
  ├── N8N/Windmill (автоматизации)
  ├── Telegram-бот
  └── Syncthing ←→ локальный Obsidian

Доступ: Termius с любого устройства
VPN: Keenetic маршрутизация для конкретных доменов
```

### OpenClaw (ex-ClaudeBot) — FOS #19
- Telegram-бот с доступом ко всей системе
- GLM 4.7 ($140/год, безлимит) или Claude
- Telegram-группа с тредами = UI (каждый тред — проект)
- Уровни реактивности по тредам

### Agent-to-Agent — FOS #19
- **Murmur** (Signal API): шифрованный P2P между ботами
- Боты решают задачи между собой, эскалируют на человека
- Фильтры на входящие (prompt injection protection)

### Агентная сеть — FOS #21
- 37 агентов + 16 MCP-серверов
- Jarvis (Claude Code) раздаёт задачи
- GitHub Projects как доска для агентов
- Murmur 2 для межагентной коммуникации

### Детерминированное + AI — FOS #19
На одного бота вешается и 100% автоматизация (сохранит ссылку), и AI-агент (ответит на вопрос).
