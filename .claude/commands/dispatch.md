Создай GitHub Issue для AI-агента на основе задачи: $ARGUMENTS

## Инструкция

Ты диспетчер задач. Получил задачу от пользователя. Нужно создать GitHub Issue и назначить правильного агента.

### Шаг 1 — Определи агента

Выбери агента по типу задачи:

| Задача | Агент | Лейбл |
|--------|-------|-------|
| Написать код, добавить фичу | Jules | `jules` |
| Сложный рефакторинг, архитектура | Codex | (упомянуть @codex в теле) |
| Баг-фикс | Jules | `jules` |
| Документация, README, комментарии | Jules | `jules` |

По умолчанию используй Jules.

### Шаг 2 — Сформируй Issue

Создай Issue через gh CLI:

```bash
gh issue create \
  --repo NickStr11/cortex \
  --title "<чёткое название задачи до 60 символов>" \
  --label "jules" \
  --body "<подробное описание что нужно сделать, критерии готовности>"
```

Body Issue должен содержать:
- Что нужно сделать (2-3 предложения)
- Acceptance criteria (список что должно работать)
- Контекст если нужен (какие файлы затронуть)

### Шаг 3 — Подтверди

После создания выведи:
- Номер и ссылку на Issue
- Какой агент назначен
- Что делать дальше (ждать PR от Jules)

## Пример

Задача: "добавить логирование ошибок в модуль dispatch"

Результат:
```
✓ Issue #5 создан: https://github.com/NickStr11/cortex/issues/5
✓ Агент: Jules (лейбл jules)
✓ Jules получит задачу и создаст PR через несколько минут
```
