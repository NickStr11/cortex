Запусти мета-обучение: проанализируй паттерны работы и предложи улучшения.

## Инструкция

### Шаг 1 — Собери данные

1. Прочитай DEV_CONTEXT.md — историю изменений
2. Посмотри последние 20 коммитов:
   ```bash
   git log --oneline -20
   ```
3. Посмотри какие файлы чаще всего менялись:
   ```bash
   git log --pretty=format: --name-only -20 | sort | uniq -c | sort -rn | head -15
   ```
4. Посмотри открытые и закрытые Issues:
   ```bash
   gh issue list --state all --limit 20
   ```

### Шаг 2 — Найди паттерны

Определи:
1. Какие типы задач повторяются? (баги, фичи, рефакторинг, доки)
2. Какие файлы — hotspots (меняются чаще всего)?
3. Какие агенты справляются лучше? (Jules vs Codex)
4. Есть ли задачи которые стоит автоматизировать?

### Шаг 3 — Предложи улучшения

На основе паттернов предложи:
1. Новые slash-команды которые автоматизируют повторяющиеся действия
2. Новые хуки для предотвращения частых ошибок
3. Улучшения в workflow (council → dispatch → agent → PR)
4. Файлы-hotspots которые стоит разбить или рефакторить

### Шаг 4 — Вывод

```

```
