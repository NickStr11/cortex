Test-Driven Development воркфлоу для задачи: $ARGUMENTS

## Инструкция

### Шаг 1 — Пойми задачу

1. Прочитай описание задачи: `$ARGUMENTS`
2. Определи что именно нужно реализовать
3. Определи какие файлы будут затронуты

### Шаг 2 — Напиши тесты ПЕРВЫМ

Создай тесты ДО написания кода:

1. Определи тестовый фреймворк (pytest для Python, vitest/jest для JS/TS)
2. Напиши 3-5 тестов покрывающих:
   - Happy path (основной сценарий)
   - Edge cases (граничные случаи)
   - Error cases (обработка ошибок)
3. Запусти тесты — они ДОЛЖНЫ падать (RED)

### Шаг 3 — Напиши минимальный код

1. Напиши минимум кода чтобы тесты прошли
2. Не оптимизируй, не рефактори — только чтобы GREEN
3. Запусти тесты — все должны пройти (GREEN)

### Шаг 4 — Рефактори

1. Улучши код сохраняя тесты зелёными
2. Убери дупликацию
3. Запусти тесты ещё раз — всё ещё GREEN

### Шаг 5 — Отчёт

```
TDD REPORT
==========
Task: [описание]
Tests written: X
Tests passing: Y/X
Files created: ...
Files modified: ...

Cycle: RED → GREEN → REFACTOR ✓
```
