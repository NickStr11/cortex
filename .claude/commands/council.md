Сгенерируй план задач через AI-консилиум на основе PROJECT_CONTEXT.md: $ARGUMENTS

## Инструкция

Ты AI-консилиум из 4 ролей: **CPO**, **CTO**, **CMO**, **Growth**.
Твоя цель — превратить контекст проекта в конкретный backlog задач для агентов.

## Спец-режим: Forge Port Sprint (обязателен)

Если в аргументах есть «claude-forge», «forge», «портировать» или «best features»,
включи режим **Forge Port Sprint**:

1. Сначала кратко перечисли 5-8 сильных практик/фич из `claude-forge` (или ближайшего аналога, если репозиторий недоступен).
2. Выбери **3-5** фич для переноса в Cortex в ближайший спринт.
3. Для каждой выбранной фичи создай отдельную задачу в backlog.
4. Каждая задача должна включать явный scope переноса: какие файлы Cortex менять и какой артефакт получится.
5. Если внешний репозиторий недоступен, явно укажи это и используй safe-default: перенос паттернов из mature AI-template (agents, hooks, slash-commands, quality gates, handoff/memory, security checks).

### Шаг 1 — Прочитай контекст

1. Открой `PROJECT_CONTEXT.md`.
2. Открой `DEV_CONTEXT.md`.
3. Учти аргументы пользователя (`$ARGUMENTS`) как приоритет CEO.

### Шаг 2 — Сгенерируй задачи по ролям

Сформируй минимум по 1 задаче от каждой роли:

- **CPO**: ценность для пользователя, UX, приоритизация фич
- **CTO**: архитектура, инфраструктура, качество кода, automation
- **CMO**: дистрибуция, контент, позиционирование
- **Growth**: метрики, эксперименты, воронка, удержание

Для каждой задачи укажи:
- `title` (до 60 символов)
- `owner` (`Jules` или `Codex`)
- `size` (`S`/`M`/`L`)
- `impact` (`Low`/`Medium`/`High`)
- `description` (2-3 предложения)
- `acceptance_criteria` (3-5 пунктов)
- `cortex_files` (список файлов/папок, которые нужно изменить)

### Шаг 3 — Отфильтруй и приоритизируй

1. Удали дубликаты.
2. Оставь только задачи, которые реально можно отдать в GitHub Issue.
3. Отсортируй по приоритету: сначала высокий impact при малом размере.
4. В финальном списке должно быть 5-8 задач.
5. В режиме Forge Port Sprint минимум 3 и максимум 5 задач должны быть прямо про перенос фич.

### Шаг 4 — Подготовь к /dispatch

Выведи результат в формате:

```markdown

### Executive Summary

- Главный риск: ...
- Фокус недели: ...

### Forge Findings
- Доступ к repo: есть/нет
- Ключевые фичи/практики: ...
- Выбрано к переносу в Cortex (3-5): ...

### Prioritized Backlog

   - Role: CPO/CTO/CMO/Growth
   - Owner: Jules/Codex
   - Size: S/M/L
   - Impact: High/Medium/Low
   - Description: ...
   - Cortex files: ...
   - Acceptance Criteria:
     - [ ] ...
     - [ ] ...

### Ready for /dispatch
- [ ] <title 1> (owner: Jules/Codex, size: S)
- [ ] <title 2> (owner: Jules/Codex, size: M)
```

### Правила качества

- Без воды: только конкретные, исполнимые задачи.
- Никаких задач без критериев готовности.
- Не предлагай задачи, требующие секретов или платных сервисов без альтернатив.
- Если контекста недостаточно — сначала перечисли, каких данных не хватает, и предложи safe-default план.
