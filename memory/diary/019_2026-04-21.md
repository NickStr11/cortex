### 1. VoiceType — диагностика «завис, пишет 0 секунд»
Симптом: постоянные start→stop через 14мс, 0 bytes аудио. Жалоба связывалась с новым USB-микрофоном (Fifine K669B).

- Добавлял audit-hook через `sys.addaudithook` для трассировки спавна процессов — не сработало, значит child появляется ДО импорта main.
- **Ключевой вывод:** `.venv\Scripts\pythonw.exe` это launcher-shim uv venv на Windows, всегда спавнит base Python (`C:\...\Python312\pythonw.exe`) как child. Два процесса — это один логический VoiceType. main() выполняется в child (видно по одному `Config loaded` в логе на каждый запуск).
- Убил hook, вернул main.py в исходное. VoiceType работает.
- Default микрофон в Windows — уже USB PnP (новый Fifine), открыл `mmsys.cpl,,1` + `ms-settings:sound` для настройки уровней.

Реальная причина залипания в более ранних логах (ошибка `SetForegroundWindow` + быстрый цикл) осталась без фикса — отложено до воспроизведения.

### 6. apteka-bot — escape_markdown фикс
Симптом: команда «История» → бот шлёт только «Необработанные (dates): 15 шт.» без списка карточек. Раньше шёл список.

Нашёл код в `/opt/apteka-bot/src/bot/telegram_bot.py:720-748`. Логика правильная — цикл шлёт по карточке на заказ. В journalctl нашёл traceback:
```
telegram.error.BadRequest: Can't parse entities:
can't find end of the entity starting at byte offset 268
```
Карточки слались с `parse_mode="Markdown"`, в `order.products` (название препарата) попадался символ из `* _ [ ] ( ) \``, Telegram находил незакрытый entity, exception прерывал цикл **на первой же итерации** — шапка «15 шт.» до того успевала.

Фикс: добавил `from telegram.helpers import escape_markdown` + обернул все пользовательские поля в `escape_markdown(str(x), version=1)`. 

Путь к фиксу был с граблями:
- Первая попытка Python-replace сломала import на `from telegram.helpers import escape_markdown, KeyboardButton, ...` (replace зацепил «хвост» импорта). Бот упал с `ImportError`.
- CRLF line endings в файле мешали второму replace матчить.
- В итоге скачал локально, нормализовал LF, применил Edit-тулом точечно, залил обратно, рестартнул сервис.

Сервис active, 15 карточек должны теперь приходить корректно.

## Что узнал

- uv venv на Windows = launcher-shim: child process с base pythonw — это норма, не «дубликат». wmic показывает разные CommandLine и ExecutablePath, но main() один раз.
- Codex часто делает больше, чем ожидается в брифе — перед новой параллельной задачей **сначала читать код**, потом планировать. Сегодня из 7 пунктов брифа все 7 были уже в коде.
- `escape_markdown(text, version=1)` — правильный способ экранировать user-контент при `parse_mode="Markdown"`. `version=2` для `MarkdownV2` с другими правилами.
- Serial console output в gcloud (`get-serial-port-output --port=1`) видит kernel OOM-killer даже когда ssh умер. Незаменимо для диагностики зависших инстансов.
- Gemini Saved Info жёстче цензурит фарму в инструкциях — триггерится на «доза», «рецепт», «отпуск». Обходится перефразировкой в нейтральный язык.
