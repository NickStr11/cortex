### VoiceType — от идеи до рабочего продукта за сессию

**Предыстория:** Кент скинул ссылки на agency-agents и mem0. Обсудили — для personal use overkill. Потом тестировал Superwhisper ($8.50/мес), понравилось качество. Решили сделать свой аналог бесплатно.

**GSD full cycle:**
- /gsd:new-project → 4 параллельных ресёрчера (Stack, Features, Architecture, Pitfalls)
- Requirements: 14 штук, 2 фазы
- Phase 1 (Foundation): scaffold, config, audio recorder (sounddevice 16kHz mono), hotkey (pynput → Win32 RegisterHotKey), window capture (ctypes). 4 плана, 3 волны.
- Phase 2 (Live Pipeline): STT (Chirp 3, us multi-region, ru+en), LLM cleanup (Gemini 2.5 Flash via Vertex AI), paste (pywin32 SetForegroundWindow + Shift+Enter для TG), tray icon (pystray, 3 состояния). 3 плана, 2 волны.

**Фиксы по ходу:**
- pynput GlobalHotKeys ненадёжен на Windows → переписан на Win32 RegisterHotKey API
- Gemini API key auth не работает с service account → Vertex AI backend
- Multi-language STT не доступен в us-central1 → Chirp 3 в "us" region с ru-RU + en-US
- Audio >60s → chunking по 55s (Recognize API лимит)
- Ctrl+Alt+R проскакивал символ → сменили на Ctrl+Shift+Space
- .env переменная HOTKEY vs VOICETYPE_HOTKEY — рассинхрон
- Shift+Enter вместо Enter для Telegram (не отправляет сообщение)
- asyncio Queue RuntimeError при Ctrl+C — cosmetic, не фиксили

**Дополнительные фичи:**
- Fast mode (regex cleanup, 0ms) — toggle в трее
- Local mode (Ollama qwen2.5:1.5b) — 0.3s, без сети. Качество пока слабое, модель добавляет отсебятину.
- Autostart — VBS скрипт в Windows Startup
- Restart.bat на рабочем столе — одним кликом перезапуск
- Улучшенный fast_clean: разбивка на абзацы по смысловым маркерам, перечисления с новой строки, группировка по 4 предложения
